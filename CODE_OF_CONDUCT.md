# Code of Conduct

## Our standard

KISS PM should be a constructive, technical, and respectful project space.

Expected behavior:

- be direct without being hostile;
- criticize ideas and code, not people;
- assume good intent until there is evidence otherwise;
- keep discussions relevant to the project;
- avoid sharing private, confidential, or security-sensitive information publicly.

Unacceptable behavior:

- harassment, threats, or personal attacks;
- discriminatory language or conduct;
- publishing private information without consent;
- public disclosure of unresolved security vulnerabilities;
- spam or bad-faith participation.

## Enforcement

Maintainers may remove content, close issues, reject pull requests, or block accounts that violate these expectations.

If you need to report conduct that should not be public, contact the maintainer through their public GitHub profile and ask for a private channel.
