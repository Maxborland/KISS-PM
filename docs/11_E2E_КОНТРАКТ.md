# 11. E2E-контракт и качество

## Почему E2E важно

Ценность KISS PM — не в отдельных функциях, а в полных управленческих контурах. Поэтому E2E является источником правды для state-changing flows.

## Паттерн приемки write-flow

Для важного действия E2E должен доказать:

1. Пользователь видит стартовое состояние.
2. Authorized user выполняет действие.
3. Unauthorized или out-of-scope user не может выполнить действие.
4. UI показывает новое состояние.
5. API/persisted state изменилось корректно.
6. Audit/action log создан.
7. Связанная surface обновилась.
8. Reload сохраняет результат.

Toast или screenshot без проверки состояния не является приемкой.

## Слои тестов

- Unit tests: domain engines, formulas, scheduling, permissions, resource calculations.
- Integration tests: API/application services, repositories, transactions.
- E2E smoke: критический путь приложения.
- E2E phase: обязательные сценарии текущей фазы.
- E2E regression: полный набор перед релизом.

## Обязательные E2E journey

1. Tenant admin настраивает роль, поле, KPI или surface.
2. CRM user создает opportunity и запускает intake.
3. Resource manager проверяет capacity и резервирует емкость.
4. Project manager создает draft и активирует проект.
5. Пользователь открывает Gantt, меняет задачу, видит audit.
6. Executor работает через My Work.
7. Resource overload выявляется и разрешается через preview/action/audit.
8. KPI deviation создает control signal и corrective action.
9. Проект закрывается, snapshot попадает в retrospective.
10. Retrospective insight меняет будущий template или threshold.

## Правило фаз

Фаза не завершена, если ее mandatory E2E отсутствует, skipped, flaky или проверяет только видимость кнопки без доменного результата.

## Нельзя

- Ослаблять тесты ради прохождения.
- Считать screenshot заменой E2E.
- Считать unit tests доказательством полного управленческого потока.
- Принимать state-changing feature без audit evidence.
