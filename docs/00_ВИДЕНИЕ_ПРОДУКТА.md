# 00. Видение продукта

## Суть

KISS PM — это SaaS/self-hosted платформа для операционного управления проектами, ресурсной загрузкой и управленческим контролем.

Название означает **Keep It Simple, Sonny Project Manager**. Это не шутка в брендинге, а правило продукта: обычный пользователь должен видеть простую последовательность действий, а сложность должна жить внутри доменной модели, расчетов, прав, аудита и E2E-проверок.

## Что продукт делает

KISS PM соединяет в один контур:

- CRM-вход и коммерческую возможность;
- оценку спроса по ролям, стадиям и срокам;
- проверку ресурсной емкости;
- создание проектного черновика;
- активный проект, Gantt, задачи и назначения;
- ресурсную матрицу и выявление конфликтов;
- KPI, отклонения и control signals;
- governed actions из управленческих поверхностей;
- аудит решений;
- закрытие проекта и ретроспективное обучение;
- улучшение шаблонов будущих проектов.

Главный путь:

```txt
CRM opportunity -> оценка спроса -> проверка емкости -> project draft -> active project -> Gantt/tasks/resources -> KPI/control signal -> governed action -> audit -> closure -> retrospective -> improved templates
```

## Продуктовый закон

Каждый важный сигнал в системе должен быть объяснимым, actionable, permission-checked, auditable и полезным для будущего планирования.

Если экран только показывает данные и не помогает понять риск, открыть контекст, принять решение или выполнить разрешенное действие, он неполон.

## Не строим

- Не строим BitrixReports-клон.
- Не строим набор статических отчетов.
- Не строим BI-конструктор вместо management platform.
- Не строим hardcoded-систему под одну компанию.
- Не строим таск-трекер без Gantt, ресурсов, KPI и аудита.
- Не строим MS Project-клон как продукт, UI или import/export в первой итерации, но Phase 5/6 backend сразу закладывает MS Project-class planning core для scheduling, зависимостей, календарей, constraints, Work/Duration/Units, critical path, ресурсов и scenario proposals.
- Не переносим старый код автоматически.

## Критерий успеха

Продукт готов к первой серьезной версии только когда E2E-доказан полный контур: tenant-настройка, CRM-вход, проект, Gantt, задачи, ресурсы, KPI, действие, аудит, закрытие и ретроспективное улучшение шаблонов.

## Главная продуктовая ставка

Сильные стороны BR2 сохраняются как возможности: матричная загрузка, мультипроектный контроль, интерактивность, несколько путей выполнить действие, подсказки, автозаполнения, выявление и решение ресурсных конфликтов. Но все это переосмысляется как универсальная SaaS/self-hosted платформа, а не как Bitrix-specific отчетность.
