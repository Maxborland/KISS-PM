## Замечания пользователя (2026-07-04, ручной осмотр)

### BUG-011 — /login: форма не по центру
- **Факт:** окно входа смещено, не отцентрировано по вертикали/горизонтали. Ожидание — карточка входа по центру вьюпорта.
- **Проверить в фазе 4:** контейнер-обёртка `(auth)/layout` или surface — отсутствует `min-h-screen`/`place-items-center` либо конфликт с design-v3 foundation-версткой.

### BUG-012 — Гант не на полный экран
- **Факт:** на /projects/[id]/schedule область диаграммы Ганта не занимает доступную ширину/высоту — визуально «окошко» вместо полноэкранного рабочего поля.
- **Связано:** плотность данных QA-инвентаря по /schedule (30 позиций) — узкая область усугубляет читаемость.

### BUG-013 — сырые технические метки в UI
- **Факт (подтверждено смоуком, BUG-007):** в /projects колонка «Спрос» показывает `position-engineer` вместо «Инженер»; пользователь также видит `project.demand` (имя поля/тип демэнда из GET) и непереведённый «дэмэнд» в интерфейсе.
- **Причина-кандидат:** UI рендерит сырые id/типы вместо резолва через справочник позиций (`GET /api/workspace/positions` / users) и словарь локализации; ср. с projects-part-A BUG-PROJ-07 (PR видит сырые id).

### BUG-014 — прототип/демо-блоки в проде
- **Факт:** на боевых роутах остаются баннеры «Прототип», «Демо-прототип», «contract-mock … данные in-memory», «v0.1 · прототип», демо-креды на /login. Данные при этом реальные из Postgres — маркеры лгут в обратную сторону и должны быть удалены/скрыты в прод-сборке.
- **Сводит:** BUG-005, BUG-AUTH-08, BUG-SHELL-15, прототип-баннеры CRM (agent-отчёт), Comms (COMM-баннеры), Admin (ADM-баннеры). Это ОДНА первопричина — общий флаг/обёртка прототипа, не снятая для прод-режима. Кандидат на связный фикс в фазе 4.

## BUG-001 — /login: нет редиректа после входа

- **Шаги:** открыть http://127.0.0.1:3000/login → email `admin@kiss-pm.local`, пароль `admin12345` → «Войти».
- **Ожидание:** редирект в рабочую область (/dashboard или /my-work).
- **Факт:** остаёмся на /login, экран «Вы вошли в KISS PM… В приложении здесь — переход в рабочую область» + кнопка «Выйти».
- **Доказательства:** network: `POST /api/auth/login → 200`, `GET /api/auth/me → 200`; снапшот 2026-07-04T11:00 (.playwright-mcp/page-2026-07-04T11-00-22-525Z.yml).
- **Заметка:** сессия при этом рабочая — прямой заход на /dashboard авторизован.

## BUG-002 — shell: сайдбар не навигирует

- **Шаги:** войти → /dashboard → навести/кликнуть любой пункт сайдбара («Проекты», «Сделки», …).
- **Ожидание:** переход на соответствующий роут (роуты существуют: /projects, /crm/deals, …).
- **Факт:** элементы не `<a>`, тултип «Демо-прототип: навигация подключится в рабочем приложении»; футер сайдбара «v0.1 · прототип».
- **Доказательства:** a11y-снапшоты /dashboard и /projects от 2026-07-04 (refs f1e10-f1e27, f2e10-f2e27).

## BUG-003 — shell: поиск в топбаре disabled

- **Шаги:** любой авторизованный роут → поле «Найти задачу или ресурс».
- **Факт:** `textbox [disabled]` — контрол мёртвый. По стандарту проекта (sb-vs-prod): мёртвых контролов быть не должно.

## BUG-004 — favicon 404

- **Шаги:** открыть любую страницу.
- **Факт:** `GET /favicon.ico → 404`, ошибка в консоли на каждой навигации.

## BUG-005 — устаревшие прототип-бейджи на боевых страницах

- **Шаги:** /login, /dashboard, /projects.
- **Факт:** бейдж «Прототип … Транспорт — contract-mock … данные in-memory», при этом фактический транспорт — реальный API (:4010, данные из Postgres seed). Маркеры прототипа не соответствуют действительности и вводят в заблуждение.
- **Заметка:** по стандарту репо честность маркеров prototype-vs-real — обязательна; здесь маркер «врёт» в обратную сторону.

## BUG-006 — /projects: нет перехода в проект

- **Шаги:** /projects → клик по строке «Портал подрядчиков Вектор».
- **Ожидание:** переход на /projects/project-vektor-portal (роут и вкладки существуют).
- **Факт:** строка — не ссылка, перехода нет.

## BUG-007 — /projects: сырой id позиции в «Спрос»

- **Факт:** «position-engineer · 160 ч» вместо «Инженер · 160 ч».

## BUG-008 — /login: мёртвые ссылки register/password-reset

- **Факт:** «Создать аккаунт» и «Забыли пароль?» помечены «Демо-прототип: навигация подключится…», при этом страницы /register и /password-reset существуют в app router.

## BUG-009 — /projects/[id]: коммуникации проекта запрашиваются для чужого id

- **Шаги:** войти админом → открыть /projects/project-vektor-portal (любую вкладку с виджетами коммуникаций).
- **Факт (network):** `GET /api/workspace/conversations?entityType=project&entityId=proj-portal → 404`, аналогично meetings и call-rooms — `proj-portal` захардкожен, реальный id `project-vektor-portal` не подставляется.
- **Следствие:** обсуждения/встречи/звонки проекта никогда не загрузятся ни для одного реального проекта.

## BUG-010 — сценарии: 403 для админа — ⚪ АРТЕФАКТ ОКРУЖЕНИЯ

- Первый смоук шёл через web :3000 → API :4010 из worktree `v3-backend-unblock`, у которого `KISS_PM_TRUSTED_MUTATION_ORIGINS=http://127.0.0.1:3001` → все мутации с origin :3000 получали 403 `same_origin_action_required`.
- На чистом стенде (:3000→:4020, текущий чекаут): login 200, logout 200, `POST …/scenarios/preview` с пустым телом → 400 `planning_scenario_invalid` (корректная валидация). Не баг продукта.
- См. env-diff.md, раздел про чужие процессы.
