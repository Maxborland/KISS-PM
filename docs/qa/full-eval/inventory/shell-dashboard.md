# Инвентарь: Оболочка, дашборд, «Моя работа», настройки, профиль, агент

Область: `/` (корень), `/dashboard`, `/my-work`, `/settings`, `/profile`, `/agent`; общая оболочка
`apps/web/src/shell/**` (app-shell, app-sidebar, app-topbar, command-palette, topbar-breadcrumbs),
`apps/web/src/app/error.tsx`, `apps/web/src/app/not-found.tsx`, прод-каркас `apps/web/src/delivery/ui/workspace-shell.tsx`.

Роли (легенда):
- **А** — админ (`admin@kiss-pm.local` / `admin12345`)
- **И** — инженер (`engineer@kiss-pm.local`)
- **ЧП** — читатель плана (`plan-reader-no-resources@kiss-pm.local`)
- **НР** — наблюдатель ресурсов (`resource-reader@kiss-pm.local`)
- **все** — все четыре роли; **аноним** — без сессии.

Важно: прод-роуты используют каркас `WorkspaceShell` (`delivery/ui/workspace-shell.tsx`), а НЕ компоненты
`src/shell/**`. `src/shell/**` (AppShell/AppSidebar/AppTopbar/CommandPalette/TopbarBreadcrumbs) в прод-роутах
не смонтированы — они используются только в `views/layout/workspace-chrome.tsx` и Storybook-витринах.
Позиции по ним включены (это заявленная область), но помечены «(SB-хром)».

## Замечания по коду

1. **[Баг-кандидат, критично] Дашборд: CRM-данные на прод-роуте идут из мока.**
   `apps/web/src/app/dashboard/page.tsx` оборачивает `DashboardSurface` только в `WorkspaceRuntimeProvider live`.
   Но `DashboardSurface` использует `useCrm()` (`apps/web/src/crm/lib/use-crm.ts`), который читает транспорт из
   `useCrmRuntime()` (`apps/web/src/crm/lib/crm-runtime.tsx`, default `{ live: false }`). `CrmRuntimeProvider` на
   роуте НЕ смонтирован → на боевом `/dashboard` плитки «Открытые сделки», «Сделки выиграны» и карточка «Сделки
   по статусам» считаются из `createMockCrmFetch()` (in-memory демо-данные), а задачи/проекты — из реального API.
   Это классический «фейковый контрол»: реальные и фейковые числа смешаны на одном экране. Также из-за этого
   ветка `forbidden` (DASH-009) на проде фактически недостижима корректно. Fix: добавить `<CrmRuntimeProvider live>`.

2. **[Баг-кандидат] Навигация прод-оболочки не работает.** `delivery/ui/workspace-shell.tsx`: все пункты сайдбара —
   неинтерактивные `<span title="Демо-прототип…">`, глобальный поиск — `<input disabled>`, аватар захардкожен «КБ».
   При этом бейджи пунктов захардкожены («12», «8», «37», «42») — фейковые числа, противоречат политике честности
   («не фейковые числа»). Роуты `/projects`, `/crm/*`, `/communications/*`, `/admin/*` существуют, но добраться до них
   из оболочки нельзя — только прямым URL. Ролевого скрытия пункта «Администрирование» нет.

3. **[Баг-кандидат] CommandPalette не навигирует.** `shell/command-palette.tsx`: пункты «Дашборд/Проекты/Сделки»
   в `onSelect` только закрывают диалог — перехода нет. Компонент смонтирован только в SB-хроме
   (`views/layout/workspace-chrome.tsx`), на прод-роутах командной палитры нет вовсе (Ctrl+K в проде не работает).
   Хоткей-обработчик не проверяет `event.target` — перехватывает Ctrl+K даже из полей ввода.

4. **[Баг-кандидат] Тема/акцент из профиля не применяются.** `PATCH /api/profile/theme` сохраняет `theme`/`accentColor`
   (profile-surface.tsx), но никто не вызывает `setTheme` из `next-themes` и не прокидывает `accentColor` в CSS-переменные;
   `app/providers.tsx` фиксирует `defaultTheme="light" enableSystem={false}`. Контрол «работает» (200 от API), но видимого
   эффекта нет — кандидат «фейковый контрол» по критерию наблюдаемого результата.

5. **[Баг-кандидат] /agent: статическая «История» и демо-хром на прод-роуте.** `agent-surface.tsx` собран на компонентах
   `widgets/landing-agent-demo/components.tsx`: `AgentConversationList` рендерит захардкоженные `HISTORY_ITEMS`
   (история запусков — фейк), `CollapsedAppNav` — декоративная навигация лендинга, времена сообщений синтетические
   (`10:41 + n минут`). Ядро (propose/stream, execute, вложения) — боевое, обёртка — демо.

7. **[Замечание] Корень `/` — мёртвая заглушка** («KISS PM — design-v3 foundation», текст про «Phase 1») без
