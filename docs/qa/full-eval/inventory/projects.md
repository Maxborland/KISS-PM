# Инвентарь: Проекты

Область Full Product Evaluation Loop — самая большая продуктовая область: список проектов, карточка проекта и 9 поверхностей Project Delivery.

**Роуты (Next.js app router, `apps/web/src/app/projects/**`):**

| Роут | Surface-компонент | Транспорт |
|---|---|---|
| `/projects` | `ProjectsListSurface` (`workspace/projects`) | `WorkspaceRuntimeProvider live` → `GET /api/workspace/projects` |
| `/projects/[id]` | `ProjectDetailSurface` (`workspace/project-detail`) | live → `GET /api/workspace/projects/:id` |
| `/projects/[id]/overview` | `ProjectOverview` (`delivery/overview`) | `PlanningRuntimeProvider live` → planning read-model + audit-events |
| `/projects/[id]/schedule` | `ProjectSchedule` (`delivery/schedule`) | live → planning read-model + apply-command(-batch) |
| `/projects/[id]/resources` | `ProjectResources` + `ResourceLoadMatrix` | live → read-model (`resourceLoad`) |
| `/projects/[id]/assignments` | `ProjectAssignments` | live → read-model + assignment.* команды |
| `/projects/[id]/baseline` | `ProjectBaseline` | live → read-model (`baselineComparison`) + baseline.capture |
| `/projects/[id]/calendars` | `ProjectCalendars` | live → read-model (`calendars`,`calendarExceptions`) |
| `/projects/[id]/scenarios` | `ProjectScenarios` | live → scenarios/preview + scenarios/:id/apply |
| `/projects/[id]/commits` | `ProjectCommits` | live → `GET /api/tenant/current/audit-events?projectId=…` |
| `/projects/[id]/settings` | `ProjectSettings` | live → read-model + project.deadline.move |

**Базовые API planning:** `GET/POST /api/workspace/projects/:id/planning/{read-model, preview-command, apply-command, apply-command-batch, scenarios/preview, scenarios/:sid/apply, baselines}`. Все мутации — паттерн **preview→apply** с `clientPlanVersion` (409 `plan_version_conflict` при рассинхроне) и аудит-событием.

**Тестовые роли (легенда колонки «Роли»):**

- **A** — `admin@kiss-pm.local` / `admin12345` — все права;
- **PR** — `plan-reader-no-resources@kiss-pm.local` / `reader12345` — только `tenant.projects.read` + `tenant.project_plan.read`: читает план, **не должен** видеть персональные календарные исключения ресурсов (`includeResourceExceptionsFor`), все мутации → 403;
- **RR** — `resource-reader@kiss-pm.local` / `resource12345` — только `tenant.project_resources.read`: planning read-model → **403 forbidden** (нет `project_plan.read`), т.е. ВСЕ вкладки `/projects/[id]/*` на read-model обязаны показать состояние «Нет прав» (`SurfaceState forbidden`), а `/projects` — ошибку доступа.

Статус всех позиций: **⬜ не проверено**.

> **Часть A прогнана 2026-07-04** (роуты `/projects`, `/projects/[id]`, `/overview`, `/schedule` — PROJ-001…060, PROJ-117): баги в `../bugs/projects.md` (BUG-PROJ-01…16). Часть B (resources/assignments/baseline/calendars/scenarios/commits/settings) — за другим агентом.

## Замечания по коду

**Использование mock-данных реальными страницами (баг-кандидаты «фейковый контрол / фейковые данные»):**

1. **Мок-справочник `RESOURCES` в редакторах на live-роутах.** `apps/web/src/delivery/lib/planning-demo-data.ts` → статический `RESOURCES` импортируют: `schedule-editors.tsx` (ResourceEditor — ячейка «Ресурсы», TaskModal — селект «Исполнитель»), `assignments-editors.tsx` (AddAssigneeDialog), `resources-editors.tsx` (ReserveDialog/AbsenceDialog), `schedule-surface.tsx` (optimisticPatch). На прод-роутах это предлагает НЕсуществующих сотрудников и отправляет мок-`resourceId` в `assignment.upsert` / `calendar.exception.upsert`. При этом «правильный» механизм уже есть — `useResourceDirectory()` (live → `GET /api/workspace/users`) — и используется в assignments-инспекторе/календарях для отображения. Требуется миграция всех селектов на `resDir.list`.
2. **Мок-эпоха времени.** `isoToDay/dayToIso` из `planning-demo-data` привязаны к `2026-03-02`; `schedule-surface` хардкодит «Сегодня» = `2026-04-28`, `overview-surface` — `TODAY = "2026-06-23"` (просрочки), `calendars-surface` — `BASE_MS = 2026-03-02` (сетка месяцев), `resources-editors` — дефолтные даты `2026-04-06/17`.
