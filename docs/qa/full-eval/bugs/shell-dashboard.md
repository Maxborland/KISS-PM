## BUG-SHELL-04 — КРИТИЧНО: CRM-плитки дашборда считаются из мока, а не из боевого API

- **Серьёзность:** критическая (фейковые бизнес-числа на боевом экране; смешение реальных и мок-данных)
- **Шаги:** 1) Войти админом → /dashboard. 2) Сравнить плитки «Открытые сделки», «Сделки выиграны» и карточку «Сделки по статусам» с GET /api/workspace/opportunities. 3) Повторить под ЧП (нет CRM-прав) и beta (другой tenant).
- **Ожидание:** числа/суммы = данным боевого CRM; для ЧП — ветка forbidden «Доступ к CRM ограничен».
- **Факт:**
  - дашборд показывает «Открытые сделки 10 / 20,6 млн ₽», «Сделки выиграны 1», статусы «Новые 5, Проверка 1, Готовы 4, Выиграны 1» — это данные `createMockCrmFetch()`;
  - реальный API: 6 сделок, статусы {project_created:3, feasibility:1, new:1, ready_to_activate:1}, открытая сумма 4 080 000 ₽, выигранных 0;
  - network-лог страницы: /dashboard вызывает только GET /api/workspace/my-work и /projects — GET /api/workspace/opportunities НЕ вызывается вовсе;
  - ЧП: GET /api/workspace/opportunities → 403 permission_missing, но дашборд показывает те же мок-числа (ветка forbidden недостижима);
  - beta (другой tenant): те же мок-числа.
  - Причина (замечание №1 инвентаря подтверждено): `DashboardSurface` → `useCrm()` без `CrmRuntimeProvider live` → mock-транспорт.
- **Доказательства:** `bug-shell-04-admin.png`, `bug-shell-04-plan-reader.png`; network: только my-work+projects; API-сверка в логе прогона.

## BUG-SHELL-12 — КРИТИЧНО: /agent — propose всегда «Предложений нет.» (model: mock-llm), ядро агента на стенде нефункционально

- **Серьёзность:** критическая (главный контрол страницы не даёт результата ни при каких входах)
- **Шаги:** 1) Войти админом (полный набор прав) → /agent. 2) Отправить «Передвинь плановый финиш задачи „Тестирование портала" на 3 дня позже». 3) Отправить «Найди перегрузки ресурсов и предложи, что сделать» (в сиде есть просроченные задачи и назначения).

- **Доказательства:** сырой SSE-ответ в логе прогона; POST /api/workspace/agent/propose/stream → 200 c `"model":"mock-llm"`.

## Примечания (не оформлено багом)

- SET-001: активная вкладка /settings не сохраняется в URL (нет `?tab=`) — минор.
- PROF-005: логаут работает («Вы вышли из системы», «Войти снова» → честный forbidden), но со страницы forbidden нет ссылки на /login — попадание на форму входа только прямым URL.
- MYWORK: переходы статусов работают в обе механики: select — 409 «Отклонено: Переход между этими статусами не разрешён» с откатом значения, разрешённый переход — «Статус изменён…»; DnD — PATCH 200 и переезд карточки. Просроченный срок подсвечен danger-красным.
- Матрица уведомлений: PUT /api/workspace/notification-preferences → 200, полный upsert подтверждён.
- Вложения агента: POST /api/workspace/attachments/files → 201, чип, «×» убирает; без якоря — подсказка, файл-пикер не открывается.
