# Инвентаризация UI-поверхностей — UI/UX loop 2026-07-05

37 роутов app router + глобальный shell. Детальную инвентаризацию контролов/состояний внутри группы делает агент группы (его выход — controls-level inventory + вердикты).

## Acceptance-критерии UI/UX (применяются к каждой поверхности)

Поверхность считается прошедшей, если по каждому пункту нет нарушения:

1. **Удобство**: основной сценарий поверхности выполняется без тупиков; ключевое действие достижимо ≤3 кликов от входа на поверхность; формы сохраняют ввод при ошибке.
2. **Логичность**: навигация и заголовки соответствуют содержимому; действия ведут туда, куда обещают; терминология единая (RU, без EN/RU-мешанины); состояние после действия отражается сразу (без ручного refresh).
3. **Достаточность функционала**: CRUD-цикл замкнут (создать→увидеть→изменить→удалить, где применимо); у списков есть поиск/фильтр/сортировка при >10 записях; у сущности есть детальная страница/просмотр. «Скоро»/заглушка при существующем API-эндпоинте — автоматический fail (стандарт репо).
4. **Кривость**: нет визуального джанка (скачки layout, обрезанный текст, наложения, непрокликиваемые зоны), нет сырых ID/техстрок/`Invalid Date`/epoch-дат в UI, loading- и error-состояния оформлены.
5. **Недоделанность**: нет мёртвых кнопок/ссылок (клик без эффекта), нет placeholder-текстов и mock-данных, все заявленные в навигации разделы существуют; пустые состояния объясняют «что это и как начать».
6. **Обратная связь**: успех/ошибка действия видимы (toast/inline); ошибки валидации по месту и на русском; деструктивные действия имеют подтверждение.
7. **Доступ по роли (G8)**: отказ в доступе — внятный экран, не белая страница/сырой JSON/бесконечный спиннер; недоступные действия скрыты или задизейблены с объяснением.

## Edge-cases (risk-based, конечный список на группу)

- Пустые данные (новый тенант/роль без прав) и большие списки (сид-данные).
- Невалидный/чужой ID в URL (`/projects/nonexistent`, `/crm/deals/999`).
- Двойной сабмит формы; сабмит с пустыми/граничными значениями.
- Прямой заход по deep-link без предварительной навигации.
- Узкий вьюпорт 1280×800 и 1024×768 (десктоп-минимум).
- Отказ зависимого сервиса: video provider disabled (G5).
