# Лендинг kisspm.app — косяки на мобилке

Зафиксировано пользователем 2026-07-05 (скриншоты iPhone, kisspm.app). Исходник лендинга — вне этого репо (отдельный reg.ru деплой, см. память `kisspm-app-beta-deploy`); чинить там же.

## 1. Автофокус на поле ввода героя — убрать
- **Экран:** IMG_5102. При загрузке страницы поле «Опишите цель или попросите…» получает фокус → на мобиле сразу открывается клавиатура и перекрывает весь первый экран.
- **Ожидание:** без autofocus; клавиатура открывается только по тапу пользователя.
- **Фикс:** снять `autoFocus` с hero-инпута (и/или не фокусировать программно на маунте).

## 2. Кнопка «Сверка» — подсветить как следующий шаг
- **Экран:** IMG_5103. Агент отвечает «…Подготовил сверку из 5 изменений», но непонятно, куда жать дальше. Кнопка «Сверка» висит в шапке справа невыразительной ссылкой.
- **Ожидание:** после ответа агента «Сверка» должна читаться как явный CTA (акцентная кнопка/подсветка/пульс), чтобы вести пользователя к результату.
- **Фикс:** выделить «Сверка» акцентным стилем (как primary), возможно появление/подсветка после ответа агента.

## 3. Блок «6 шагов» — на всю высоту экрана
- **Экран:** IMG_5104 (шаг «01 · CRM · Новая сделка»). Сейчас шаг не занимает высоту вьюпорта — сверху видна шапка следующего шага, снизу торчит пунктирный коннектор.
- **Ожидание:** каждый из 6 шагов = один экран по высоте (`min-h-[100svh]`), желательно со scroll-snap по шагам.
- **Фикс:** обёртка шага `min-h-[100svh]` + `scroll-snap-align: start` на контейнере секции (использовать `svh`, не `vh` — учесть мобильные адрес-бары).

## 4. (bonus) Горизонтальный оверфлоу строки ввода героя
- **Экран:** IMG_5102. Синяя кнопка отправки обрезана правым краем — строка «инпут + скрепка + отправить» шире вьюпорта.
- **Фикс:** строка ввода `max-width: 100%` / `flex-wrap`/усадка инпута, чтобы кнопка отправки помещалась.
