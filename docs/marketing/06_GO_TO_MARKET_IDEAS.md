# Идеи go-to-market

## Стадия

KISS PM находится на стадии до публичного запуска / закрытая альфа. Главная цель сейчас - проверить, что agent-first project management и proposed diff review действительно цепляют целевую аудиторию, а не звучат как AI hype.

## Основной GTM-motion

Рекомендуемый первый motion:

> Founder-led outreach к проектным и операционным лидерам с демонстрацией одного workflow: цель -> агент -> project diff -> review -> apply -> audit.

Этот motion подходит стадии, потому что категория требует объяснения, а самый важный proof - не список функций, а доверительный demo.

## Приоритетные идеи

| Приоритет | Идея | Почему подходит | Первые шаги |
|---:|---|---|---|
| 1 | Demo-first alpha outreach | Показывает отличие быстрее, чем текст | Собрать список PM/COO/PMO, отправить короткое сообщение с GIF или скрином diff |
| 2 | Видео "задержка -> project diff" | Делает концепцию конкретной | Записать 90-секундный walkthrough с agent prompt и hunks |
| 3 | Founder essay series | Строит категорию без generic AI | 3 текста: agent-first PM, why diff, why audit |
| 4 | Fake-door призыв `Посмотреть project diff` | Проверяет интерес к центральной метафоре | Призыв на лендинге ведет к заявке или запросу демо |
| 5 | Серия интервью | Добывает язык целевой аудитории | 15 интервью по последнему replanning / meeting-to-task случаю |
| 6 | Comparison page tests | Ловит demand от пользователей AI PM tools | Сравнить KISS PM с AI-chat-in-PM подходом |
| 7 | Prototype walkthrough | Проверяет trust | Показать Figma/HTML mock и смотреть, где люди начинают доверять или сомневаться |

## Сообщения для личного outreach

### Короткая версия

> Мы строим KISS PM как agent-first project management: пользователь задает цель, агент готовит project diff, а менеджер применяет только одобренные изменения. Хотим показать ранний прототип и понять, узнаете ли вы в этом свою project routine.

### Через боль

> После встреч, задержек и клиентских изменений PM часто вручную обновляет задачи, сроки, владельцев и статусы. Мы проверяем другой workflow: агент готовит изменения как diff, а человек ревьюит и применяет выбранное.

### Для технической аудитории

> Если Cursor/Codex сделали reviewable diffs привычным способом работать с кодом, можно ли такой же trust-паттерн перенести в project management? Мы строим KISS PM вокруг этой идеи.

### Для консервативной B2B-аудитории

> KISS PM не дает AI молча менять проект. Агент готовит предложения, пользователь утверждает, система сохраняет before/after и audit trail.

## Стратегия сообщества

На ранней стадии лучше собрать небольшой экспертный круг вокруг identity:

> Люди, которые хотят, чтобы AI помогал двигать проекты вперед, но не менял проект без человеческого контроля.

Форматы:

- закрытый Telegram/Slack для alpha advisors;
- monthly teardown: один реальный replanning или meeting-to-task кейс;
- private notes о том, как проектные агенты должны проходить approval;
- разборы "где AI PM tools звучат сильнее, чем работают";
- анонимные lessons from alpha calls.

Цель community:

- customer language;
- trust objections;
- demo feedback;
- design partners;
- referrals.

## Типы co-marketing партнеров

| Тип партнера | Почему подходит | Идея кампании |
|---|---|---|
| PMO consultants | Видят governance и ручные replanning loops | Вебинар: `AI в PM без silent automation` |
| CRM / delivery integrators | Видят gap между обещанием клиенту и проектной реальностью | Guide: `от встречи к project diff` |
| Agile / delivery coaches | Работают с rituals, status, handoff | Workshop: `как meeting notes превращать в approved changes` |
| Resource planning consultants | Понимают последствия сроков и ролей | Demo: `resource-aware project diff` |
| Партнеры по self-hosted / инфраструктуре | Доверие, данные, governance | Статья: `почему AI-действиям нужен audit trail` |

## Позиция по платному привлечению

Платный трафик стоит тестировать только после того, как есть:

- понятный hero;
- видимый project diff demo;
- baseline conversion;
- повторяющиеся боли целевой аудитории;
- language from interviews.

Если тестировать:

- LinkedIn по ролям: project manager, PMO lead, COO, head of delivery, resource manager;
- search ads по запросам: `AI project management`, `project replanning`, `meeting notes to tasks`, `project management approval workflow`;
- retargeting только после просмотров diff demo.

## Материалы перед публичным запуском

1. Лендинг с agent conversation + project diff hero.
2. 90-секундное demo video.
3. Founder essay: `Почему AI не должен молча менять проект`.
4. Alpha request form.
5. One-page PDF: `Agent-first project management без потери контроля`.
6. Чеклист: `Можно ли доверять AI-агенту проектные изменения?`
7. Short deck для discovery calls.

## Метрики alpha GTM

Отслеживать:

- qualified alpha requests;
- долю заявок, где человек упоминает ручное обновление после встреч;
- долю заявок, где важен approval/audit;
- клики на `Посмотреть Сверку`;
- booked conversations;
- completed discovery calls;
- повторяющиеся objections;
- willingness to test with a real project;
- реакцию на metaphor "Cursor for project management".

Первый GTM win - не количество лидов, а повторяемое доказательство: люди понимают diff-review workflow и называют его решением своей реальной рутины.
